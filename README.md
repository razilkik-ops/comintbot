# Comint Telegram Sales Bot на Node.js

Это MVP Telegram-бота продаж для comint.by без конструктора ботов.

Что уже есть:

- webhook для Telegram Bot API;
- свободный ввод клиента;
- распознавание услуг по синонимам и ошибкам: `банер`, `стикеры`, `визиточки`, `наружка`, `ролап`;
- стартовая сегментация клиента: полиграфия, сувениры, сложный проект;
- кнопочная воронка полиграфии: тип продукции, тираж, срочность;
- кнопочная воронка сувениров: тип сувенира, событие, бонус на первый заказ;
- сценарии вопросов по услугам;
- более мягкий сценарий общения с подсказками и прогрессом по вопросам;
- команда `/services` и кнопка `Услуги`;
- полный каталог услуг с `https://comint.onrender.com/services` в `data/catalog.json`;
- команда `/prices` и кнопка `Цены` с утверждённым прайсом на типовые позиции;
- команда `/status` и кнопка `Статус заявки` для клиента;
- отмена текущей заявки через `/cancel` или кнопку `Отменить заявку`;
- предварительный расчёт типовых позиций из утверждённого прайса;
- приём файлов от клиента;
- отправка заявки и файлов в Telegram-группу менеджеров;
- команда менеджера `/reply LEAD_ID текст`, чтобы ответить клиенту через бота;
- команды менеджера `/lead`, `/status`, `/done` для просмотра и обновления заявок;
- хранение заявок в `data/leads.json`;
- хранение текущих диалогов в `data/sessions.json`.

## 1. Что понадобится

1. Telegram-бот, созданный через BotFather.
2. Токен бота.
3. Хостинг/VPS, где можно запускать Node.js.
4. HTTPS-домен или поддомен, например `https://bot.comint.by`.
5. Telegram-группа менеджеров.

## 2. Установка на сервере

Скопируйте папку проекта на сервер.

В папке проекта выполните:

```bash
npm install
```

Скопируйте файл `.env.example` в `.env`:

```bash
cp .env.example .env
```

Откройте `.env` и заполните:

```env
BOT_TOKEN=токен_из_BotFather
PORT=3000
PUBLIC_BASE_URL=https://bot.comint.by
WEBHOOK_PATH_SECRET=длинная_секретная_строка
TELEGRAM_SECRET_TOKEN=еще_одна_секретная_строка
MANAGER_CHAT_ID=
COMPANY_NAME=Коминт
SALES_MANAGER_NAME=Екатерина
FIRST_ORDER_DISCOUNT=5
```

Секретные строки можно придумать вручную или сгенерировать:

```bash
node -e "console.log(require('crypto').randomBytes(24).toString('hex'))"
```

## 3. Запуск

Для проверки:

```bash
npm start
```

Проверка в браузере:

```text
https://bot.comint.by/health
```

Должно вернуться:

```json
{"ok":true,"service":"comint-telegram-sales-bot"}
```

## 4. Подключение webhook

После того как Node.js приложение доступно по HTTPS, выполните:

```bash
npm run set-webhook
```

Проверить webhook:

```bash
npm run webhook-info
```

Удалить webhook:

```bash
npm run delete-webhook
```

## 5. Как получить MANAGER_CHAT_ID

1. Создайте Telegram-группу менеджеров.
2. Добавьте бота в группу.
3. Желательно дать боту права администратора.
4. Напишите в группе:

```text
/chatid
```

Бот ответит примерно так:

```text
chat_id этого чата:
-1001234567890
```

Скопируйте это значение в `.env`:

```env
MANAGER_CHAT_ID=-1001234567890
```

Перезапустите Node.js приложение.

## 6. Каталог услуг и цены

Каталог сайта хранится в:

```text
data/catalog.json
```

Бот объединяет этот каталог со сценариями из:

```text
data/services.json
```

На `https://comint.onrender.com` числовые цены не опубликованы, поэтому `data/prices.json` оставлен с нулевыми значениями. Пока значения равны `0`, бот не показывает неподтверждённые суммы и передаёт заявку менеджеру на индивидуальный расчёт.

## 7. Как настроить цену баннера

Откройте файл:

```text
data/prices.json
```

Найдите:

```json
"banner_m2": {
  "value": 0,
  "unit": "м²",
  "comment": "Цена за 1 м² на сайте не опубликована. При 0 бот передаст расчёт менеджеру."
}
```

Замените `0` на актуальную цену за 1 м².

Например:

```json
"banner_m2": {
  "value": 20,
  "unit": "м²",
  "comment": "Цена за 1 м² баннера"
}
```

После изменения перезапустите приложение.

## 8. Как настроить стартовые цены для полиграфии

В файле:

```text
data/prices.json
```

есть блоки для стартовых цен стандартной полиграфии:

```text
business_cards_standard
flyers_standard
stickers_standard
```

Каждый блок содержит диапазоны тиража:

```text
up_to_100
100_500
500_1000
over_1000
```

Сейчас значения `value` равны `0`, потому что сайт не публикует числовой прайс. Замените их на актуальные стартовые цены только после получения подтверждённого прайса. Если клиент выберет срочный заказ, бот применит `urgentMultiplier`.

## 9. Как добавить новую услугу

Откройте:

```text
data/services.json
```

Добавьте новый объект по аналогии:

```json
{
  "code": "calendar",
  "category": "Календари",
  "name": "Календари",
  "synonyms": ["календарь", "календарики", "домик", "квартальный календарь"],
  "intro": "Понял, нужны календари.",
  "managerOnly": true,
  "askFile": true,
  "calc": { "type": "manager" },
  "questions": [
    { "key": "calendar_type", "label": "Тип календаря", "prompt": "Какой календарь нужен: домик, квартальный, настенный, карманный или другой?" },
    { "key": "quantity", "label": "Тираж", "prompt": "Укажите тираж." },
    { "key": "deadline", "label": "Срок", "prompt": "К какому сроку нужен заказ?" }
  ]
}
```

Важно: `services.json` должен оставаться корректным JSON. После правки можно проверить:

```bash
node -e "JSON.parse(require('fs').readFileSync('data/services.json','utf8')); console.log('OK')"
```

## 10. Как менеджеру ответить клиенту

Когда бот отправит заявку в группу, внизу будет строка:

```text
/reply 20260520-ABC123 текст сообщения
```

Менеджер пишет в группе, например:

```text
/reply 20260520-ABC123 Добрый день! Для точного расчёта уточните, пожалуйста, материал баннера.
```

Бот отправит это сообщение клиенту в личный чат.

Дополнительные команды менеджера:

```text
/lead 20260520-ABC123
```

Показывает заявку повторно в группе.

```text
/status 20260520-ABC123 в работе
```

Обновляет внутренний статус заявки. Клиент увидит этот статус через `/status`.

```text
/done 20260520-ABC123
```

Отмечает заявку как выполненную.

Можно сразу отправить клиенту финальное сообщение:

```text
/done 20260520-ABC123 Заказ готов, можно забирать сегодня до 18:00.
```

## 11. Команды клиента

```text
/services
```

Показывает список услуг, которые бот умеет обрабатывать.

```text
/prices
```

Поясняет, что числовые цены на сайте не опубликованы, и предлагает оформить заявку для индивидуального расчёта.

```text
/status
```

Показывает последнюю заявку клиента в текущем Telegram-чате.

```text
/manager
```

Запускает быструю передачу заявки менеджеру.

```text
/cancel
```

Отменяет текущий незавершённый диалог.

## 12. Где хранятся заявки

Заявки лежат здесь:

```text
data/leads.json
```

Текущие незавершённые диалоги:

```text
data/sessions.json
```

Для MVP этого достаточно. Если заявок станет много, лучше заменить JSON-хранилище на SQLite/PostgreSQL.

## 13. Nginx reverse proxy пример

Если приложение работает на `localhost:3000`, а наружу должен смотреть поддомен `bot.comint.by`, можно проксировать через Nginx:

```nginx
server {
    server_name bot.comint.by;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

SSL можно выпустить через Let's Encrypt.

## 14. Запуск через PM2

Для постоянной работы можно использовать PM2:

```bash
npm install -g pm2
pm2 start src/index.js --name comint-telegram-bot
pm2 save
pm2 startup
```

После изменения `.env`, `services.json` или `prices.json` перезапуск:

```bash
pm2 restart comint-telegram-bot
```

## 15. Что важно по ограничениям

Этот бот работает бесплатно, потому что не использует платный AI API и платный конструктор.

Но это не полноценный ChatGPT-бот. Он понимает запросы через:

- синонимы;
- похожие слова;
- нечёткое сравнение;
- заранее заданные сценарии вопросов.

Для первого рабочего бота продаж этого достаточно. Если потом понадобится более «умное» понимание сложных фраз, можно добавить LLM: OpenAI API, локальную модель или другую AI-интеграцию.
